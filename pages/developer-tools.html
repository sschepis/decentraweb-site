Developer Console & Platform Tools

Developer tools for building, testing, and monitoring on the DecentraClick platform: GraphQL console, contract deployment, event explorer, block explorer, usage metering, and service monitoring.

Developer Console

Accessible at /developer, the Developer Console provides a unified interface for all development tasks:

  • Contract Deployment Wizard — Deploy platform contracts on supported networks via CLI or the web-based wizard at /contract-deploy. Includes network selection, owner configuration, step-by-step progress tracking, and final deployment summary. Supports CREATE2 deterministic deployment via DynamicContractDeployer
  • Compilation Server — WebSocket-based Solidity compiler (port 7770). Dual-engine: Forge (primary) + solc.js (fallback). Compiles and returns ABI + bytecode in real-time. Token-authenticated with path traversal protection
  • Diamond Management — Visual diamond proxy manager at /diamond-manage: deploy diamonds, add/remove facets, configure permissions, view facet selectors, call any facet function

Event Explorer

Browse and filter on-chain events from platform contracts. The EventExplorerPage (/events-explorer) and EventExplorerApp provide:

  • Real-Time Events — Live event streaming via the Event Bus with on-chain bridging. All 30+ event channels available
  • Filtering — Filter by contract address, event name, block range, and indexed parameters
  • Detail Inspection — View full event payload, transaction context, decoded log data, and block metadata
  • Export — Export events as CSV/JSON for external analysis

Block Explorer

On-chain block explorer at /block-explorer (also available as the BlockExplorerApp in the App Store):

  • Block Browser — Browse latest blocks with timestamp, transaction count, gas used, and miner info
  • Transaction Lookup — Search by transaction hash to view sender, recipient, value, gas, status, and event logs
  • Address Pages — View address balances, transaction history, and contract interactions
  • Multi-Chain — Switch between supported networks (Sepolia, OP Sepolia, OP Mainnet, Base)

Usage Metering & Billing

The platform tracks API usage for billing and rate limiting:

MetricDescription
API CallsGraphQL queries, mutations, and subscription events tracked per-user per-domain
Compute UsageWASM function execution time and resource consumption
Data StorageGun.js data volume, IPFS storage, and DDB record counts
BandwidthCDN bandwidth, d-web traffic, and peer sync data transfer

View usage reports at /usage with aggregation by day, week, or month. Usage data is collected by the serviceMetering and usageMeteringService modules.

Service Monitoring

The ServiceMonitorApp and /service-monitoring page provide:

  • Health Dashboard — Real-time status of all registered services with uptime percentages, latency charts, and error rates
  • Alert Configuration — Set thresholds for latency, error rates, and downtime. Alerts fire via the Event Bus and can trigger webhooks
  • Metrics Collection — serviceMetricsCollector aggregates uptime, latency, and throughput from serviceDiscovery scans
  • Issue Tracking — Auto-creates issues on service degradation with severity classification and resolution tracking

Contract Compiler

The Compilation Server (port 7770) compiles Solidity contracts submitted via WebSocket:

// Connect to compilation server
const ws = new WebSocket('ws://localhost:7770');

// Authenticate
ws.send(JSON.stringify({ type: 'auth', token: COMPILATION_TOKEN }));

// Submit compilation
ws.send(JSON.stringify({
  id: 'req-1',
  source: '// SPDX-License-Identifier: MIT\npragma solidity ^0.8.0;\n...',
  contractName: 'MyContract'
}));

// Receive result
ws.onmessage = (event) => {
  const result = JSON.parse(event.data);
  // result.contract = { name, source, bytecode, abi }
};
