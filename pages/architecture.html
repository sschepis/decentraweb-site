Architecture Overview

How DecentraClick's layers work together to deliver a sovereign, composable platform.

System Architecture

DecentraClick is organized into four distinct layers, each with clear responsibilities and well-defined interfaces between them.

Layer 1: Presentation

The UI layer is built with React 18 and TypeScript, using Vite for fast development. It includes:

  • 59 Pages — Lazy-loaded routes for every platform feature, from domain management to DeFi strategies
  • 109 Components — Reusable UI building blocks organized by domain (auth, blockchain, diamond, identity, etc.)
  • 13 Custom Hooks — React hooks that bridge UI to services (useDAO, useCompute, useGraphQL, usePWA, etc.)
  • Tailwind CSS — Utility-first styling with theme support (light/dark/custom themes)

Layer 2: Application Logic

Business logic lives in Zustand stores and service modules, providing a clean separation from the UI.

  • 45 Zustand Stores — Persistent state management with domain-scoped isolation
  • 71 Service Modules — Core business logic for every feature (identity, DAO, compute, DeFi, etc.)
  • Event Bus — Unified pub/sub across local, P2P, and on-chain event sources with 30+ typed channels
  • Plugin SDK — Sandboxed extension system with capability-based permissions

Layer 3: Data & Network

Data flows through multiple decentralized channels depending on the use case.

  • Gun.js — P2P database for real-time, conflict-free data with offline support and SEA encryption
  • IPFS — Content-addressed storage for DWeb sites, NFT metadata, and large files
  • GraphQL — In-browser API server with 6 schema modules (core, wallet, defi, apps, infra, social)
  • Namespace Service — Multi-tenant data isolation at the domain, platform, and user scope levels

Layer 4: Blockchain Settlement

On-chain state is managed through a comprehensive smart contract system on Optimism.

  • 36+ Registry Contracts — On-chain registries for domains, identities, services, apps, roles, and more
  • Diamond Proxy (EIP-2535) — Modular contract architecture with 43+ facets for composable functionality
  • ERC-4337 Account Abstraction — Smart accounts with paymaster gas sponsorship and session keys
  • Multi-network — Deployable to Optimism mainnet, Optimism Sepolia, and Ethereum Sepolia

Key Design Principles

  • Domains as Organizations — Every domain is a full organizational unit with its own members, roles, treasury, and governance
  • Composability — Features are modular and can be mixed via Diamond facets, plugins, and service composition
  • Sovereignty — Data lives on P2P networks and blockchain, not centralized servers
  • Offline-First — PWA with service workers, offline sync queues, and Gun.js local-first replication
  • Multi-Tenant Isolation — Namespace service ensures data isolation between domains, users, and the platform
