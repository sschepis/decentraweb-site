Account Abstraction (ERC-4337)

Smart accounts with gasless transactions, paymasters, session keys, and bundled operations.

Overview

DecentraClick implements ERC-4337 Account Abstraction, enabling smart contract wallets that abstract away gas payment, support batched transactions, and provide enhanced security features like guardian recovery and session keys.

Core Components

  • Smart Account — A contract-based wallet that replaces EOA limitations with programmable logic
  • Entry Point — The singleton contract that validates and executes UserOperations
  • Bundler — Aggregates multiple UserOperations into a single on-chain transaction
  • Paymaster — Sponsors gas costs for users based on configurable policies
  • Account Factory — Deploys new smart accounts with deterministic addresses

Paymaster Policies

Organizations can configure gas sponsorship policies for their members:

PolicyDescriptionUse Case
Free GasOrganization pays all gasOnboarding new users
Token PaymentUsers pay gas in ERC-20 tokensToken-native experiences
Rate LimitedFree gas up to a daily/weekly limitFreemium models
ConditionalFree gas for specific contract interactionsIncentivizing platform usage

Session Keys

Session keys allow dApps to execute transactions on behalf of the user without requiring approval for each transaction. Keys are scoped and time-limited.

  • Time-bounded — Keys expire after a configurable duration
  • Scope-limited — Keys can only call specific contracts and functions
  • Spending limits — Maximum value per transaction and total
  • Revocable — Users can revoke session keys at any time

Transaction Batching

Smart accounts can batch multiple operations into a single UserOperation, reducing gas costs and improving UX. For example, approve + swap in a single transaction.
