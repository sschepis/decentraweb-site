Cross-Chain Identity Aggregator

Resolve identities across multiple protocols (DecentraClick, ENS, Lens, Farcaster) and normalize them into a unified, confidence-scored profile.

💡
The aggregator is initialized on app bootstrap and reacts to identity changes across all adapters. It bridges the gap between on-chain identity registries and social/web3 identity protocols.

How It Works

The Resolution Engine accepts any identifier (wallet address, ENS name, Lens handle, Farcaster FID, DecentraClick identity ID) and queries all adapters in parallel:

  1. Query Adapters — Each adapter translates the input identifier into its native query format and returns identity data
  2. Normalize — A Normalization Pipeline maps disparate schema fields into a unified AggregatedIdentity format
  3. Confidence Score — A Confidence Scorer weighs each source by reliability, data freshness, and cross-source agreement
  4. Merge Results — Field-level consensus merges data from multiple sources; conflicting fields use the highest-confidence source

Identity Normalization

All identity data is normalized into a unified AggregatedIdentity structure:

interface AggregatedIdentity {
  // Unified profile
  displayName: string;
  avatar: string;
  bio: string;

  // All known identifiers
  id: string;           // primary DecentraClick identity ID
  wallets: string[];    // all linked wallet addresses
  aliases: {            // cross-protocol handles
    ens?: string;
    lens?: string;
    farcaster?: number;
  };

  // Verifiable claims
  claims: { topic: number; issuer: string; valid: boolean }[];
  kycStatus: 'none' | 'pending' | 'verified' | 'rejected';

  // Confidence metadata
  confidenceScore: number;  // 0-100
  sourceBreakdown: { adapter: string; confidence: number }[];
  lastUpdated: number;
}

Confidence Scoring

Each resolution is assigned a 0-100 confidence score based on:

  • Source Reliability — On-chain data (DecentraClick, ENS) scores higher than social data (Lens, Farcaster)
  • Data Freshness — Recently updated profiles get higher confidence; stale data decays
  • Cross-Source Agreement — When multiple sources agree on a field (e.g., same wallet), confidence increases
  • Claim Validity — Verified KYC claims from trusted issuers boost the score significantly

Storage Backends

Resolved identities can be stored in three backends:

Gun.js

Default backend. Full aggregated identity stored in user's P2P graph for offline access and cross-device sync

IPFS

Verifiable credential storage (ipfsCredentialStore). Attestation data pinned for long-term availability

DDB

Structured identity schemas in the DecentraClick Database for queryable, schema-enforced storage

Key Components

ResolutionEngine

Orchestrates multi-adapter resolution, normalizes results, merges with confidence weighting

NormalizationPipeline

Maps each adapter's output schema to the unified AggregatedIdentity format

ConfidenceScorer

Weighted scoring based on source reliability, data freshness, and cross-source consensus

useIdentityAggregator

React hook: resolve(identifier), myAggregatedIdentity, reactive subscriptions

Page Route

The /identity-aggregator page provides a search interface where users can resolve any identifier across all protocols and view the unified profile, confidence breakdown, and per-adapter raw data.
