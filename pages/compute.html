Serverless Compute

Deploy WASM functions triggered by on-chain events, webhooks, or cron schedules.

Overview

The Compute Service provides serverless function execution within DecentraClick. Functions are written in JavaScript/TypeScript, compiled to WASM, and executed in a sandboxed environment with configurable triggers.

Trigger Types

TriggerDescriptionExample
On-Chain EventExecute when a smart contract emits an eventRun on every token transfer
WebhookExecute when an HTTP endpoint is calledProcess incoming API requests
Cron ScheduleExecute on a recurring time scheduleDaily report generation

Function Capabilities

  • WASM Sandbox — Functions run in isolated WASM environment for security
  • Memory Limits — Configurable memory allocation per function
  • Timeout Control — Maximum execution time to prevent runaway functions
  • Execution Metrics — Track invocations, duration, errors, and resource usage
  • Namespace Access — Functions can read/write namespace-scoped data
  • Event Bus Integration — Functions can publish events to the event bus

Function Lifecycle

  1. Write — Write function code in the built-in editor
  2. Configure — Set trigger, memory limits, timeout, and permissions
  3. Deploy — Register the function on-chain via ComputeRegistry
  4. Monitor — Track execution metrics and logs
