Decentralized Database (DDB)

P2P data storage powered by Gun.js with namespace isolation, CRDT resolution, and offline support.

Overview

The Decentralized Database (DDB) service provides structured data storage on top of Gun.js. Data is stored peer-to-peer with CRDT conflict resolution, ensuring eventual consistency without a central server.

Features

  • P2P Storage — Data replicates automatically across connected peers
  • CRDT Resolution — Conflicts resolved automatically using Conflict-free Replicated Data Types
  • Offline Support — Read and write data offline; syncs when connectivity returns
  • Namespace Isolation — Data scoped to domains, users, or platform level
  • SEA Encryption — User-scoped data encrypted with Gun.js SEA
  • Collections — Organize data into typed collections with schema support
  • Real-time Subscriptions — Subscribe to data changes in real-time

Data Model

DDB uses a graph-based data model where nodes can reference other nodes. Collections are organized under namespace paths.

// Example: Domain-scoped collection
platform/settings/...         // Platform-wide settings
domain/myorg/members/...      // Organization members
domain/myorg/wiki/pages/...   // Wiki pages for this domain
user/0x123.../preferences/... // User private data
