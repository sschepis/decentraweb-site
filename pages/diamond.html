Diamond Proxy (EIP-2535)

Modular smart contract architecture with 48 composable facets for tokens, DeFi, identity, governance, and stablecoins.

Overview

The Diamond Proxy pattern (EIP-2535) is the cornerstone of DecentraClick's on-chain modularity. A Diamond is a single contract address that delegates function calls to multiple implementation contracts called "facets." This allows unlimited contract size and modular upgradeability.

How Diamonds Work

  • Single Entry Point — Users interact with one contract address; the Diamond routes calls to the appropriate facet
  • Facet Selection — Each function signature maps to a specific facet contract via the Diamond Cut
  • Shared Storage — All facets share the Diamond's storage, enabling cross-facet state access
  • Upgradeable — Facets can be added, replaced, or removed without changing the Diamond address

Diamond Factory

The DiamondFactory contract allows deploying new Diamonds with a selected set of facets in a single transaction. The UI provides a visual facet selector with dependency management.

💡
Facet dependencies are automatically resolved. For example, selecting the MarketplaceFacet will automatically include ERC721AFacet if not already selected.
