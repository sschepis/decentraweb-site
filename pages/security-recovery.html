Security Dashboard & Account Recovery

Comprehensive security management: API key administration, audit logging, security scanning, and multi-method account recovery including social guardianship and biometric auth.

Security Dashboard

Accessible at /security, the Security Dashboard provides a unified interface for platform security management:

API Key Management

  • Generate API Keys — Create scoped API keys for programmatic access to platform services. Keys are stored encrypted in the user's Gun.js private space
  • Permission Scopes — Granular permissions per key: read/write access to specific namespaces, services, and domains
  • Revocation — Instant key revocation with audit trail. Revoked keys are permanently invalidated
  • Key Rotation — Scheduled key rotation with configurable expiry windows

Audit Logging

  • PII-Stripped — All audit logs are automatically scrubbed of personally identifiable information before storage
  • Comprehensive Events — Tracks: logins, contract interactions, service access, API key usage, role changes, KYC verification attempts
  • Queryable — Filter by event type, time range, IP address, or action with pagination support
  • Export — Export audit logs as CSV or JSON for external compliance tooling

Security Scanning

  • Service Scans — Automated vulnerability scanning of registered services and endpoints
  • Score Reports — Each scan produces a security score with categorized findings (critical, high, medium, low)
  • Scheduled Scans — Configurable scan schedules for continuous monitoring

Account Recovery

DecentraClick supports multiple recovery methods to prevent permanent loss of access:

Social Recovery (M-of-N Guardians)

💡
Social recovery is implemented in the SmartWallet contract via the GuardianRecoveryFacet on the Diamond proxy.
  1. Configure Guardians — Set up to N trusted guardians (other addresses) and a recovery threshold M
  2. Initiate Recovery — Request recovery to a new wallet address. Guardians are notified
  3. Guardian Approval — M of N guardians must approve the recovery request within the timeout window
  4. Execute Recovery — Once the threshold is met, ownership transfers to the new wallet. The old wallet is permanently disabled

Biometric Recovery (DFNS WebAuthn)

For users using the DFNS WebAuthn managed wallet mode, recovery leverages platform biometrics (fingerprint, Face ID) plus backup email. The DFNS service provides enterprise-grade key recovery without exposing private keys to the application.

Smart Wallet Recovery (ERC-4337)

ERC-4337 smart accounts include built-in recovery via the GuardianRecoveryFacet. The SmartWallet contract maintains a list of guardians and a recovery threshold. When recovery is initiated, a timelock ensures guardians have time to veto malicious attempts.

Data Backup & Portability

Backup Service

Export all Gun.js data (domains, apps, settings, profiles) as an encrypted backup bundle. Restore on any device

Data Portability

Export identity claims, linked wallets, and domain ownership proofs as portable credentials at /data-portability

Backup Registry

On-chain BackupRegistry anchors backup hashes for verification of data integrity

Scheduled Backups

Configure automated backups to Gun.js peers or IPFS with configurable intervals
