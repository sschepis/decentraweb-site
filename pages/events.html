Unified Event Bus

A typed pub/sub system that unifies local, P2P, and on-chain events across 30+ channels.

Overview

The Event Bus is the nervous system of DecentraClick. It provides a single, consistent API for publishing and subscribing to events regardless of their source — whether they originate locally in the browser, from P2P peers via Gun.js, or from on-chain smart contract events.

Event Sources

SourceTransportLatencyPersistence
LOCALIn-memory pub/subImmediateSession only
GUNGun.js P2P networkNear real-timeReplicated across peers
CHAINOn-chain contract eventsBlock confirmationPermanent on-chain

Key Features

  • 30+ Typed Channels — Pre-defined channels for domains, identity, contracts, services, governance, and more
  • Middleware Pipeline — Intercept and transform events with logging, filtering, and validation middleware
  • Wildcard Subscriptions — Subscribe to patterns like domain.* to catch all domain-related events
  • Request/Reply — Synchronous request-response patterns over the event bus
  • Offline Buffering — Gun.js events are buffered locally and synced when peers reconnect
  • Event History — Browse historical events through the Event Explorer app

Event Channels

Events are organized into typed channels. Some of the key channels include:

domain.*

Domain registration, updates, member changes

identity.*

Identity creation, claims, attestation updates

contract.*

Deployments, function calls, events

service.*

Service lifecycle, health, metrics

governance.*

Proposals, votes, execution

compute.*

Function deployment, execution, results

eventBus

Core event bus service with pub/sub, middleware, and multi-source routing (39KB)

eventSyncService

Synchronization between local, Gun, and chain event sources (15KB)
