Multi-Tenant Namespaces

Data isolation and scoping across domains, users, and the platform via Gun.js namespace paths.

Overview

The Namespace Service provides multi-tenant data isolation on top of Gun.js. Every piece of data in DecentraClick lives within a namespace scope, ensuring organizations can't accidentally access each other's data.

Namespace Scopes

ScopePath PatternAccessUse Case
PLATFORMplatform/{collection}Global, read by allSystem configuration, shared registries, public data
DOMAINdomain/{domainId}/{collection}Domain members onlyOrganization data, settings, member content
USERuser/{userId}/{collection}Owner onlyPersonal data, preferences, private keys

How It Works

  • Path-based isolation — Each scope translates to a unique Gun.js path, so data is physically separated in the P2P graph
  • RBAC enforcement — The namespace service checks RBAC permissions before granting access to any scoped data
  • Collection-level control — Access can be controlled at the collection level within each scope (e.g., only admins can write to the "settings" collection)
  • Encryption — User-scoped data is encrypted with Gun.js SEA (Security, Encryption, Authorization)
  • Offline support — Namespace data syncs via Gun.js CRDT resolution when peers reconnect

Service Integration

All platform services use the namespace service for data storage. When a service reads or writes data, it specifies the namespace scope, and the service routes to the correct Gun.js path automatically.

💡
The useNamespace React hook provides a convenient API for components to read and write namespace-scoped data with automatic scope resolution.
