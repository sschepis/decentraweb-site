Zero-Knowledge Attestations & Privacy Proofs

Generate and verify privacy-preserving proofs about identity claims using 4 custom ZK circuits. Prove you hold a valid attestation without revealing the underlying data.

💡
ZK proofs are generated client-side in a Web Worker using snarkjs and verified on-chain via the ZKAttestationVerifierFacet.

4 ZK Circuits (Circom 2.1.6)

All circuits use Poseidon hashing for efficient on-chain verification and are compiled with Groth16 proving. The Verifier facet is deployed as a Diamond facet.

CircuitPurposeExample Use Case
RangeProofProve a field value satisfies a threshold (gt/lt/gte/lte) without revealing the value"My credit score is above 700" — proven without sharing the score
EqualityProofProve a specific field in an attestation equals a target value"My country field is 'US'" — proven from a multi-field attestation
MembershipProofProve a value exists in a Merkle tree (up to 1024 elements)"My address is on the accredited investor list" — without revealing which one
SelectiveDisclosureReveal chosen fields from an attestation while hiding others"I'm KYC verified, and my age is over 21" — revealing 2 of 8 fields

How It Works

1. Attestation Creation

A trusted issuer (e.g., KYC provider) issues an attestation with up to 8 fields on-chain via addClaim() on IdentityRegistry. Each claim field is stored as a Poseidon commitment: commitment = Poseidon(fieldValue, salt). The attestation root is Poseidon(allFieldCommitments).

2. Proof Generation (Client-Side)

  1. Select Predicate — Choose the circuit type and configure parameters (threshold, field index, disclose mask)
  2. Resolve Inputs — predicateResolver maps user-friendly predicates to circuit inputs from on-chain attestation data
  3. Generate Witness — The Prover Web Worker runs the circuit with private inputs (field values + salts)
  4. Create Proof — Groth16 prover generates the proof bundle: { proof, publicSignals, attestationRoot }

3. On-Chain Verification

The ZKAttestationVerifierFacet on the Diamond proxy verifies proofs on-chain:

// Verify a RangeProof
function verifyRangeProof(
  uint256 threshold,
  uint256 operator, // 0=gt, 1=gte, 2=lt, 3=lte
  uint256 fieldCommitment,
  bytes32 attestationRoot,
  uint256[8] calldata proof
) external view returns (bool);

// Verify a SelectiveDisclosure
function verifySelectiveDisclosure(
  bytes32 attestationRoot,
  uint256[8] calldata discloseMask,
  uint256[8] calldata disclosedValues,
  uint256[8] calldata fieldCommitments,
  uint256[8] calldata proof
) external view returns (bool);

ZK Proof Builder UI

The platform includes a visual proof builder at /zk-proofs:

  • ZKProofBuilderPanel — Configures predicates, generates witness, and produces Groth16 proofs in-browser via Web Worker
  • ZKPredicateRow — Per-predicate configuration (range bounds, field selection, disclosure masks)
  • ZKProofResultCard — Displays generated proof bundle with verification status and option to submit to chain

Privacy Guarantees

Zero Knowledge

Verifier learns only what the predicate claims — no field values are exposed beyond disclosed fields

Non-Interactive

Proofs require no back-and-forth between prover and verifier; any third party can verify independently

Succinct

Groth16 proofs are constant-size (~128 bytes) regardless of circuit complexity

On-Chain Verifiable

The ZKAttestationVerifierFacet performs all verification in a single transaction

Key Components

zkAttestationService

Predicate resolution, proof generation dispatch, verification, Gun.js proof storage (12KB)

predicateResolver

Maps user-friendly predicates to circuit-specific inputs with field resolution

ZKProofBuilderPanel

React component for building and validating ZK proof requests in-browser

ZKAttestationVerifierFacet

Diamond facet for on-chain Groth16 proof verification of all 4 circuits
