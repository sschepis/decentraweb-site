Plugin SDK

Extend the platform with sandboxed plugins featuring 11 permission categories and iframe isolation.

Overview

The Plugin SDK allows developers to extend DecentraClick's functionality with custom plugins. Plugins run in sandboxed iframes with a capability-based permission model, ensuring platform security while enabling rich extensions.

Permission Categories

Plugins request permissions from 11 categories when installed:

CategoryDescriptionExample Scopes
StorageRead/write namespace dataread, write, delete
IdentityAccess identity informationread_profile, verify_claims
BlockchainContract interactionsread_state, send_transaction
EventsEvent bus accesssubscribe, publish
UIUI rendering capabilitiesrender_panel, show_notification
ComputeExecute compute functionsinvoke, deploy
NetworkHTTP and P2P accessfetch, gun_read
GovernanceDAO operationsread_proposals, cast_vote
FinanceFinancial operationsread_balance, create_escrow
AdminAdministrative accessmanage_members, update_settings
SystemPlatform-level accessregister_service, manage_apps

Sandbox Architecture

  • Iframe Isolation — Plugins run in sandboxed iframes with restricted capabilities
  • PostMessage Protocol — Communication between plugin and host via structured messages
  • Rate Limiting — API calls from plugins are rate-limited to prevent abuse
  • Metrics — Track plugin resource usage and performance

Plugin Lifecycle

  1. Install — Plugin is downloaded and permissions are granted
  2. Activate — Plugin initializes and registers its capabilities
  3. Run — Plugin operates within its permission sandbox
  4. Deactivate — Plugin is suspended but state is preserved
  5. Uninstall — Plugin and its data are removed
