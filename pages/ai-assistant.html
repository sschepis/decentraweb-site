AI Assistant

WebSocket-powered conversational AI agent backed by Claude, with 30+ platform tools for navigation, data access, contract interaction, and application generation.

💡
The AI Assistant runs as an overlay chat panel accessible from the navbar. Each user gets a persistent session with full conversation history and real-time streaming responses.

WebSocket Protocol

Authentication uses the user's Gun.js public key. The server maintains persistent sessions on disk at data/sessions/ with full conversation history.

MessageDirectionPurpose
authClient → ServerAuthenticate with Gun.js public key + user context
messageClient → ServerSend user text message to AI
text_deltaServer → ClientStreaming text chunk from Claude
tool_useServer → ClientTool call request from Claude (displayed in UI)
tool_resultServer → ClientResult from tool execution
client_commandServer → ClientBrowser command (navigate, open modal, etc.)
context_updateClient → ServerUpdate current page, wallet state
turn_completeServer → ClientEnd of conversation turn; AI ready for next input

Session Features

  • Persistent Sessions — Conversation history is saved to data/sessions/{publicKey}.json. Sessions survive server restarts and reconnections
  • Multi-Turn Reasoning — Up to 10 tool-calling turns per message. The AI calls tools, receives results, and reasons about them iteratively until reaching a final response
  • Real-Time Streaming — Text deltas and tool use events stream to the client as they arrive from Claude, with a typing indicator during processing
  • Offline Buffering — If the client disconnects mid-turn, messages are buffered and delivered on reconnection. Background turns continue executing
  • Auto-Compaction — When conversation context exceeds ~100k tokens, older messages are automatically summarized to stay within model limits
  • Context Awareness — The assistant knows the user's current page, domain, wallet address, and can navigate the app on the user's behalf

Key Components

assistant-server.mjs

WebSocket server (2336 lines). Session management, Claude API client, tool execution, context compaction

useAssistant

React hook for WebSocket connection, streaming chat, tool-use rendering, reconnection logic

assistantStore

Zustand store: messages[], isOpen, isConnected, isThinking, tool-uses

ChatOverlay

Floating chat panel with message list, input, and tool-use cards. Toggled from navbar
